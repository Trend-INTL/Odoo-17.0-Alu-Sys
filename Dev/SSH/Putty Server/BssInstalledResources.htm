Bitvise SSH Server: Installed resources

Components that can be used directly

The following Bitvise SSH Server components can be directly used by the administrator:

BssCtrl.exe

The SSH Server Control Panel. It provides management of server settings, host keys, and password cache. It also features an interactive SSH connections view.

BssCfg.exe

The SSH Server command-line management and configuration utility. See the Bitvise SSH Server Users' Guide (http://www.bitvise.com/ssh-server-users-guide) for usage instructions.

BssCfgManip.exe, BssCfgManip.idl

A COM object using which Bitvise SSH Server can be configured programmatically from any programming language that supports COM, including Visual Basic and VBS scripts. Open the IDL file in a text editor, like Notepad, for a specification of the interface.

bvPwd.exe

Enables a user to change their Windows password or that of another user from the Windows console. Run bvPwd -h for instructions.

bvRun.exe

Can be used to create a child process with extended CreateProcess flags, e.g. to start a child process and return immediately without waiting, or to execute the child process in a different window station and desktop, and more. Similar to start, but for more advanced types of program execution.

When used with Bitvise SSH Server, this utility provides a way to launch a program and keep it running when the SSH connection exits, using flags "-brj -new" or "-brj -det".

VirtAccountExporter.ps1, VirtAccountImporter.ps1

Example PowerShell scripts that can be used to export SSH Server virtual account settings to a file in CSV format (comma-separated values); or to import settings in CSV format. You can use the scripts as-is, or modify them to use an extended format.

uninst.exe

Bitvise SSH Server uninstaller. It is not usually run directly. Instead, use Add or remove programs in Windows.

To uninstall Bitvise SSH Server in an unattended manner:

  1. Copy uninst.exe from the Bitvise SSH Server installation directory to another temporary location.

  2. Run it in the temporary folder as follows:

    uninst "Bitvise SSH Server" -unat

    If you configured a non-default instance name for the SSH Server installation, you must pass the full instance name instead.

It is possible to run uninst directly from the SSH Server installation directory. However if you do that, uninst will first copy itself to a temporary location, then run itself and exit. The original instance of uninst will exit before the uninstallation completes. This may cause problems if your script performs other tasks which should wait for the uninstallation.

Components for use by developers

BssCfgManip.htm

A compiled HTML reference for all settings interfaces available using the SSH Server's scriptable configuration COM object. This information mirrors help available from the interfaces themselves, e.g. by executing a command such as the following in the PowerShell settings query window:

$cfg.settings.help

See also:
BssCfgManip.htm
https://www.bitvise.com/ssh-server-guide-scriptable

SfsDll.h, SfsDll32.lib, SfsDll64.lib

A C++ header and link libraries, providing an interface which an application can use to interact with the SSH Server's virtual filesystem configured for the currently logged on user. An application can access the virtual filesystem via SfsDll when run in a terminal shell, or as an exec request, under an SSH Server connection. The resources that can be accessed this way are the same resources, in the same layout, as can be accessed by the user via SFTP or SCP.

SfsDllSample.cpp, SfsDllSample.exe

A sample application that uses the SfsDll interface to access the currently logged on user's virtual filesystem. A pre-built SfsDllSample.exe is included, allowing for experimentation without having to set up a build environment. The sample needs to be run in a terminal shell, in an SSH Server connection.

BssStat.exe, BssStat.cpp

A utility that provides a way to enumerate active SSH Server connections and tunneled connections from a command line. It is accompanied by C++ source code illustrating how to access the same information from an application.

Components that should NOT be executed directly

The following Bitvise SSH Server components are invoked from within the server itself or in another special manner, and should not be launched manually:

BvSshServer.exe

The SSH Server core program. Runs as a Windows service. Launch it and stop it via the Bitvise SSH Server Control Panel, via Windows Administrative Tools > Services, or via the net stop and net start commands.

BssCtrl.ps1

The PowerShell script launched by the SSH Server Control Panel when "Query settings with PowerShell" is used. Displays help and creates the $cfg object used to query SSH Server settings in PowerShell.

BvLsaEx.dll

An authentication package installed by Bitvise SSH Server to perform password-less login when using public key authentication, as well as for virtual accounts that use a custom security context.

toterms.exe, totermh32.dll, totermh64.dll

The SSH Server's terminal subsystem. Programs executed within the terminal subsystem, including terminal shell, will have totermh32.dll or totermh64.dll loaded into their processes. Note: totermh64.dll is installed only on 64-bit Windows.

execs.exe

The SSH Server's exec request subsystem.

BvShell.exe

A Unix-like terminal shell that integrates with the SSH Server's virtual filesystem. This allows for shell access to be granted even to users who are only intended to have limited access to the filesystem. This shell can be used for commands that may not be available over SFTP or SCP. BvShell allows WinSCP to function with the SSH Server in SCP mode.

TelnetForward.exe

Bitvise Telnet forwarder utility.

AgentForward.exe

The SSH Server's authentication agent forwarding subsystem.

SfsServer.exe

The SSH Server's SFS, SFTP, and SCP subsystem.

SfsDll32.dll, SfsDll64.dll

DLLs providing an interface for applications started under an SSH connection in a terminal shell, or as an exec request, to access the virtual filesystem configured for the user in SSH Server settings. If you wish to develop an application that uses this interface, see Components for use by developers.

BssActStateCheck.exe

Starts briefly when an administrator logs in interactively to check whether the SSH Server's evaluation period is about to expire, or if it already has. If so, displays a warning message box.

BvDump32.exe, BvDump64.exe

In the event that an SSH Server process encounters a critical error, this executable is run to create a memory dump of the process that encountered the error. Any memory dumps are stored on the local computer under the current user's %LOCALAPPDATA% directory, under Bitvise\\BvDump. If you ever see an error of this type, please contact Bitvise.

CiCpFips32.dll, CiCpFips64.dll, CryptoPP530Fips32.dll, CryptoPP530Fips64.dll

The SSH Server's older cryptographic provider as used in 6.xx versions. CiCpFips32/64.dll serve as an interface to CryptoPP530Fips32/64.dll, a version of Crypto++ that originally received FIPS 140-2 validation under certificate 819.

This provider should only be used where needed. It contains known issues with timing side channels, which can permit remote private key compromise. Our software implements mitigations, but it is better to use the new provider, CiWinCng.

By default, this cryptographic provider is used on Windows XP and Windows Server 2003, where Windows CNG cryptography is not available. On later Windows versions, this provider can be selected by creating a registry value named "CryptoProvider", with string value "CiCpFips", under the registry key HKLM\Software\Wow6432Node\Bitvise on 64-bit systems, or HKLM\Software\Bitvise on 32-bit systems. This registry value will have machine-wide effect on Bitvise SSH Server, SSH Client, and FlowSsh versions 7.xx and higher.

CiWinCng32.dll, CiWinCng64.dll

The SSH Server's default cryptographic provider used on all recent versions of Windows, starting from Windows Vista and Windows Server 2008.

This provider uses Windows CNG cryptography for algorithms supported by Windows. It is augmented with OpenSSL and Crypto++ to support algorithms not supported by Windows.

If FIPS mode is enabled in Windows security policy, this provider uses only Windows CNG cryptography. In FIPS mode, only algorithms supported by Windows are available.
